# 产品介绍 — Lulu's Daily Mic

## 一、产品基础定位

**产品名称**：Lulu's Daily Mic（全球化沟通力养成助手）

**产品背景**：本产品面向雅思6.5分水平、目标全英文工作环境的 AI 产品经理。用户已具备基础英语读写能力，但口语表达不够流利、缺乏每日练习场景、缺少贴合自身水平的练习素材。市面上英语学习工具要么过于基础（少儿启蒙）、要么过于学术（雅思托福备考），缺乏「日常口语打卡 + AI PM 职业英语」这一垂直场景的产品。

**目标用户**：Vibe Coding 转型 AI 产品经理，英语雅思6.5分，需要在半年内达到全英文工作面试和工作交流水平。

**核心痛点**：
- 缺乏每日口语练习场景和固定仪式感
- 市面口语素材难度不适配（要么太简单、要么太难）
- 希望练习内容和 AI PM 职业相关，而非空洞生活话题
- 需要中文对照翻译，方便理解后跟读练习
- 需要 AI PM 学习资料体系化推送，而非碎片化阅读

**产品价值**：将「每日口语打卡」+「AI PM 职业学习」结合为一体，用 LLM 自动生成个性化口语脚本，用定时推送保持学习节奏，用网页端降低使用门槛。

## 二、完整功能清单

### 显性功能

| 功能 | 说明 |
|------|------|
| **口语脚本生成** | 输入视频/文章 URL 或主题词，自动抓取内容并生成约2分钟的英文口播脚本 |
| **固定开篇/结尾** | 每份脚本强制包含标准打卡开篇语和结束语，建立每日练习仪式感 |
| **中文对照翻译** | 每份脚本附带完整中文翻译，方便理解后跟读 |
| **多源内容抓取** | 支持 YouTube 字幕、Bilibili 字幕、网页正文、纯文本主题四种输入 |
| **历史脚本浏览** | 网页端查看全部历史脚本，支持搜索、排序、下载 |
| **AI PM 材料推送** | 每天10:00和22:00自动策展推送 AI 产品经理学习资源 |
| **macOS 原生通知** | 推送时弹出系统通知，文案生动自然，引导打开网页浏览 |
| **Web UI 界面** | Streamlit 网页端，三页面：脚本生成、脚本历史、学习材料 |
| **CLI 命令行** | 支持交互式终端操作和单次命令模式，适配定时任务触发 |

### 隐性逻辑

| 能力 | 说明 |
|------|------|
| **智能回退抓取** | 视频无字幕时自动用标题+简介生成脚本，不中断流程 |
| **词汇等级控制** | Prompt 强制使用高频2000词，禁用生僻词和复杂长句 |
| **客观转述约束** | Prompt 禁止杜撰个人经验，只客观转述学习收获 |
| **字数精准控制** | 目标250-300词，约2分钟语速 |
| **Day 序号自动递增** | 系统自动统计历史脚本数量，计算下一个 Day 编号 |

## 四、大模型选型说明

| 参数 | 配置 |
|------|------|
| **Provider** | DeepSeek |
| **Model** | deepseek-chat |
| **Temperature** | 0.7 |
| **Max Tokens** | 4096 |
| **上下文窗口** | 64K（DeepSeek-v3） |

**选型理由**：
1. **成本优势**：DeepSeek API 价格远低于 GPT-4/Claude，适合每日高频调用
2. **中文能力**：DeepSeek 中英双语能力强，脚本翻译质量好
3. **JSON Mode**：支持结构化输出，便于解析脚本和材料
4. **兼容性**：与 OpenAI SDK 接口兼容，后续可无缝切换到 GPT 或 Claude

**适配场景**：
- 英文脚本生成长度约300词，4096 tokens 输出空间充足
- 0.7 温度平衡创造力和一致性（需要严格遵守格式约束，不宜过高）
- 每日2-3次调用（1次脚本生成 + 2次材料推送），成本可控

## 五、Skill 工具能力

### ScriptSkill — 口语脚本生成

| 属性 | 说明 |
|------|------|
| **触发命令** | `script generate/list/view/delete` |
| **触发场景** | 用户提供视频URL、文章链接、或感兴趣主题 |
| **核心能力** | 抓取内容 → LLM生成250-300词英文脚本 → 附带中文翻译 |
| **输入** | URL 或 主题文本 |
| **输出** | 英文脚本 + 中文翻译 + 词数/时长元数据 |
| **使用限制** | 需网络连接（抓取外部内容）；YouTube 字幕为自动生成可能不完美 |

### MaterialSkill — AI PM 材料推送

| 属性 | 说明 |
|------|------|
| **触发命令** | `material push morning/evening`、`material list/view` |
| **触发场景** | 定时自动触发（CronCreate）或手动请求推送 |
| **核心能力** | LLM生成8篇候选 → URL可访问性HEAD/GET双重验证 → 取精准5篇有效链接；不足时自动重试（最多3次） |
| **输入** | 时段参数（morning/evening） |
| **输出** | MaterialCollection（精准5篇，每篇含标题、中文描述、类型、已验证URL） |
| **使用限制** | 依赖 LLM 在 approved sources 白名单内生成链接；HEAD/GET 验证通过不等于页面内容完全正确 |
| **URL 验证** | `_check_url()` 方法：HEAD 优先 → 失败回退 GET stream → 双重兜底确保链接可访问 |

| 调用方式 | 说明 |
|---------|------|
| **LLM 调用** | 通过 `core/llm.py` 调用 DeepSeek API（OpenAI 兼容接口），支持 chat 和 chat_json 两种模式 |

## 七、完整工作流

### 流程一：口语脚本生成

```
用户输入 URL/主题
  → WebConnector.fetch() 识别来源平台
    → YouTube: youtube-transcript-api 获取字幕
    → Bilibili: 公开 API 获取标题+描述+字幕
    → 网页: trafilatura 提取正文
    → 纯文本: 作为主题直接使用
  → 获取或计算 Day 序号 (storage.get_next_day_number)
  → 加载 prompts/script_generator.txt，替换占位符
  → LLM chat_json() 生成脚本
  → 解析 JSON → 构建 Script 对象
  → 验证固定开篇是否完整
  → storage.save_script() 持久化
  → Web UI / CLI 渲染展示
```

### 流程二：定时材料推送

```
CronCreate 触发 (10:07 / 22:07)
  → python agent.py material push morning/evening
  → MaterialSkill._cmd_push()
    → 加载 prompts/material_curator.txt（含 approved sources 白名单）
    → LLM chat_json() 策展 8 篇候选
    → _check_url() HEAD/GET 双重验证每个链接
    → 过滤取前 5 篇有效链接，不足则自动重试（最多 3 次）
    → 构建 MaterialCollection → 保存
  → macOS 弹窗通知用户
  → 用户打开网页 → 首页展示最新推送
```

### 流程三：网页浏览

```
用户打开 http://localhost:8501
  → 三标签页导航
  → 脚本生成页：输入 → 生成 → 双语展示 + 下载
  → 脚本历史页：搜索 → 浏览 → 展开全文 → 下载/删除
  → 材料页：最新推送横幅 → 材料列表 → 历史记录展开
```

## 八、提示词工程

### script_generator.txt（核心提示词）

**角色设定**：专业英语口语练习脚本撰写者。

**约束规则**：
- 固定开篇和结尾必须逐字复制，零变动
- 仅使用高频2000词汇，禁用生僻词和复杂长句
- 每句最多15个词
- 客观转述学习内容，不准编造个人经验和感受
- 输出250-300词，约2分钟语速

**输出格式**：严格 JSON，包含 english_script / chinese_translation / word_count / estimated_duration_seconds / topic

### material_curator.txt（材料策展提示词）

**角色设定**：AI 产品经理学习资源策展人。

**约束规则**：
- 每次生成精准 8 篇候选，经 URL 验证后取前 5 篇有效链接
- 必须使用 approved sources 白名单内的平台 URL（Anthropic/OpenAI/Google/Meta 官方文档、LangChain/HuggingFace/GitHub、The Verge/TechCrunch/Ars Technica、Lenny's/Pragmatic Engineer 等）
- 早间侧重可操作工具和方法，晚间侧重深度阅读和案例
- 描述用中文，语气亲切自然，像朋友推荐
- URL 必须真实可访问，不确定时只用白名单平台

**输出格式**：严格 JSON，包含 session_description / 8 篇 materials 候选

## 九、异常与边界处理

| 场景 | 处理方式 |
|------|---------|
| YouTube 视频无字幕 | 5级回退（手动英文→自动英文→任意英文→中文翻译→任意语言翻译），全部失败则阻断 LLM 生成，直接展示错误原因 |
| Bilibili API 失败 | 回退：返回降级 ContentResult，携带错误原因说明 |
| 网页抓取失败 | 回退：trafilatura 提取失败时尝试直接 HTTP 请求 |
| LLM 调用失败 | 返回错误消息，不保存任何数据 |
| JSON 解析失败 | chat_json() 内置自动清理 markdown 代码块，失败则抛出异常 |
| 固定开篇被修改 | 输出后自动检测，若开头不匹配则显示黄色警告 |
| 脚本词数超标 | Prompt 内置上限约束（max 350词），超出后提醒但不过度纠正 |
| 材料URL失效 | Prompt 要求优先使用真实URL，标注「不确定时建议如何查找」 |

## 十、迭代状态

**当前版本**：v1.0.3

**已知缺陷**：
- YouTube 部分视频仅有机器生成字幕，质量不可控
- Bilibili 字幕获取依赖视频作者上传，覆盖面有限
- youtube-transcript-api 为非官方库，版本漂移可能导致 API 兼容问题
- macOS 通知仅在本地生效，无法推送到手机

**待优化方向**：
- 增加 Telegram Bot 通道，实现手机端推送
- 增加语音合成（TTS），自动朗读生成脚本
- 增加发音评分功能
- 增加用户自定义词汇难度级别
- 接入更多内容源（播客、Newsletter）
